- [x] 1.5 Implement recording and deterministic replay of a transaction sequence (DEV-020) — every call is recorded (identity, simulated timestamp, reducer, args, commit/abort outcome); `TestShard::replay(seed, tape)` drives a fresh shard through the tape under the recorded timestamps, failing loudly at the first commit/abort divergence (`step N`); replay-equality asserted via `fingerprint()` in the suite.
- [x] 1.6 Add fault-injection hooks (mid-commit crash, torn tail) reusing the DST harness (DEV-021) — `TestShard::crash()` freezes a deterministic `kill -9` disk image (fsync watermark waited, nothing closed/checkpointed); `CrashedShard::lose_last_commit()` (entry-boundary cut = lost fsync / mid-commit crash) and `tear_last_commit()` (seeded mid-frame cut = torn write) apply the DST crash model (frozen STG-011 framing walk, same technique as `fluxum-dst::sim`); `recover()` runs the REAL checkpoint+replay recovery and hands back a live shard (recording/clock/RNG carry over). Covered: clean-crash full recovery (incl. STG-040 batched auto-inc semantics), lost-fsync drops exactly the tail, torn tail quarantined + shard keeps serving, empty-log faults are no-ops.
