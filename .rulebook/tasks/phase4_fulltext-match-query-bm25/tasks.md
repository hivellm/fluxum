- [x] 1.1 Lexer/parser: MATCH predicate over a #[fulltext] column; reject MATCH on unindexed columns (compile-time) and fuzzy ~ / OR-NOT-inside-MATCH / field-boost with 400 (FTS-030/033; sql/parse.rs) — `col MATCH 'raw'` parses to `CondAst::Match`; `FtsQuery::parse` (index/fulltext.rs) analyzes the raw string with the column's declared analyzer and rejects `~`, `^`, grouping parens, OR/NOT-inside-MATCH, non-trailing `*`, unterminated/empty phrases, and stop-word-only queries with wire-ready 400s; MATCH on a non-#[fulltext] column is an FTS-033 400 at compile
