## 1. Implementation
- [x] 1.1 Define the audit query request/response: filter by table + optional row key + time or tx_id range; each result row carries caller, reducer_name, tx_id, timestamp in commit order (OPS-020) — AuditQuery/AuditEntry in commitlog/audit.rs; POST /audit body/response in admin.rs. Prerequisite discovered and implemented: TxRecord did not carry caller/reducer_name, so they were added as tail-additive serde fields and threaded from the engine through the pipeline as CommitMeta
- [x] 1.2 Add an indexed commit-log read: map (table, row key) and time/tx_id to the segments+offsets that touched it so audit does not full-scan the log (OPS-020) — segment-granular index: a segment's filename encodes its first_tx_id and list_segments returns them sorted, so a tx window selects a contiguous slice and the rest are never opened; row matching compares a delete's logged PK and derives an insert's PK from its logged values
- [x] 1.3 Extend the read to span archived segments (local and, when configured, object-store) so history predating the live log is included — no separate audit store (OPS-020) — local retained/archived segments are covered (the read walks every segment in the log dir). Object-store archives are NOT covered: that archive does not exist yet, it lands with phase7_backup-object-storage-archive; documented in SPEC-025
- [x] 1.4 Wire the `audit` endpoint into the admin HTTP surface using the RPC-052 envelope, returning ordered results as JSON (OPS-020) — POST /audit in admin.rs (+ is_admin_path, route table)
- [x] 1.5 Access control: reject audit reads from any identity other than admin / registered server-peer (AUTH-062) with a typed error (OPS-021) — request token must authenticate to a registered server peer; client identity → 403, no credential → 401
- [x] 1.6 Apply column masking / field-crypto to audit output so masked or encrypted columns never surface plaintext in an audit result (OPS-021) — structural: the result is metadata only (tx_id, timestamp, caller, reducer_name, inserted, deleted); column values are never returned, so no plaintext can leak. Pinned by a negative assertion in the integration test
- [x] 1.7 Verification: a row changed by three reducer calls returns exactly those three entries with correct caller/tx_id/timestamp in commit order, with a non-admin caller refused — tests/audit_trail.rs drives three real reducer calls through the engine and asserts exactly those entries (an unrelated row's change excluded), plus 403/401 access control and no column value in the output. Unit tests cover a multi-segment log (live+retained boundary), tx-window pruning, whole-table trace
